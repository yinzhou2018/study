## Context

当前REPL交互模式在`src/repl.cc`中实现，使用`std::getline`读取输入，没有任何输入提示符；输出结果使用"= "前缀（第50行）。该REPL类通过`std::reference_wrapper`持有输入输出流引用，构造时可传入自定义流（默认为`std::cin`/`std::cout`），这为终端检测和测试提供了良好的扩展基础。

## Goals / Non-Goals

**Goals:**
- 在终端交互模式下显示 "> " 输入提示符
- 移除输出结果的 "= " 前缀，直接输出结果值
- 管道输入模式下不显示提示符，保持输出纯净
- 最小化对现有代码结构的改动

**Non-Goals:**
- 不实现多行输入或续行提示符（如 "... "）
- 不修改错误输出格式
- 不添加命令行参数解析（如 --no-prompt 选项）
- 不实现语法高亮或自动补全

## Decisions

### 决策1：终端检测方式

选择使用`isatty(fileno(stdin))`检测是否为终端模式。

- **选择**：`isatty()` + `fileno()`
- **备选**：在`Repl`构造函数中增加`bool interactive`参数
- **理由**：自动检测比手动传参更可靠，避免用户忘记设置；`isatty`是POSIX标准函数，macOS和Linux均原生支持。构造函数参数方式需要调用方显式传递，增加了耦合。

### 决策2：提示符存储位置

在`Repl`类中添加`bool interactive_`成员，在构造函数中通过`isatty`初始化，`Run()`循环中根据该标志决定是否输出提示符。

- **选择**：构造时检测一次，存为成员变量
- **备选**：每次循环迭代时调用`isatty`
- **理由**：终端模式在运行期间不会改变，一次检测即可；避免重复系统调用的性能开销。

### 决策3：输出前缀移除

直接将`out_.get() << "= " << result.value()`改为`out_.get() << result.value()`。

- **选择**：简单删除前缀字符串
- **备选**：使用配置化的前缀字符串
- **理由**：当前需求明确不需要输出前缀，无需过度抽象；若未来需要可配置前缀，届时再引入。

## Risks / Trade-offs

- [**BREAKING**：输出格式变更] → 已有脚本或工具若依赖"= "前缀解析输出将失败。 mitigation：在提案中标注，由使用方适配。
- [POSIX兼容性] → `isatty`/`fileno`在Windows上需`<io.h>`头文件。mitigation：当前项目仅在macOS/Linux上运行，暂不处理；如需跨平台可通过预编译宏条件包含。
